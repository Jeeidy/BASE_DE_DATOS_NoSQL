BASE DE DATOS DOCUMENTALES


Una base de datos de documentos es un tipo de base de datos NoSQL que se puede usar para almacenar y consultar datos como documentos tipo JSON. La notación de objetos de JavaScript (JSON) es un formato abierto de intercambio de datos legible tanto por los humanos como por las máquinas. Los desarrolladores pueden usar documentos JSON en su código y guardarlos directamente en la base de datos de documentos. La naturaleza flexible, semiestructurada y jerárquica de los documentos y las bases de datos de documentos permite que evolucionen según las necesidades de las aplicaciones.


Caracteristicas

Las bases de datos documentales son capaces de almacenar información en diferentes formatos sin una estructura definida. En cualquier caso, lo habitual es que los documentos empleen un formato de archivo, mientras que los datos contenidos sí utilicen una estructura fija.

A pesar de que su estructura es completamente distinta, estas bases de datos permiten realizar las mismas operaciones básicas que las bases de datos relacionales, esto es, añadir, actualizar o eliminar información, además de realizar las pertinentes consultas por parte del usuario.

A diferencia de las bases de datos relacionales, en las bases de datos orientadas a documentos no es necesario recorren todas las columnas de una tabla a la hora de realizar una consulta. En lugar de ello se asigna un identificador único a cada documento, de manera que a la hora de hacer una consulta se comprueba el mismo documento. Este identificador puede ser de diferentes tipos, por ejemplo, una ruta completa o una cadena de caracteres.

Gracias a este funcionamiento, las bases de datos documentales se emplean para el almacenamiento y consulta de datos semiestructurados, los cuáles no tienen un esquema previamente definido y que serían difíciles (o incluso imposibles) de gestionar en las tradicionales bases relacionales.

Funcionamiento

  1. Estructura Flexible: Los datos se almacenan en documentos (generalmente en formato JSON o BSON), lo que permite incluir diferentes tipos de datos y estructuras dentro del mismo documento. Por ejemplo, un documento puede contener un campo de texto, una lista, o incluso otro documento anidado.
  2. Gestión de Datos Embebidos: Los documentos pueden contener datos embebidos, lo cual facilita la gestión de datos relacionados sin necesidad de múltiples tablas o uniones complejas, como en las bases de datos relacionales.
  3. Consultas y Actualizaciones: Las bases de datos documentales permiten realizar consultas y actualizaciones sobre cualquier parte de un documento. Esto incluye la capacidad de buscar documentos basados en campos específicos y actualizar únicamente esos campos sin afectar el resto del documento.
  4. Desarrollo Ágil: Debido a su naturaleza flexible, las bases de datos documentales son ideales para entornos de desarrollo ágil y aplicaciones con requisitos cambiantes. Los desarrolladores pueden modificar la estructura de los documentos sin necesidad de realizar cambios significativos en el esquema de la base de datos.
